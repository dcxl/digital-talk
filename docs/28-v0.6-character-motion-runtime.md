# v0.6 Character Motion Runtime

## 1. 目标

v0.6 的目标是让 AI 角色在不依赖 Live2D 的前提下具备基础表现力：

- 角色图片可随运行状态产生轻量动作。
- 说话时口型、呼吸、微位移与音量联动。
- 思考、聆听、错误、中断等状态有可感知表现。
- 后续可替换为 ComfyUI 生成的视频片段、表情帧、局部动画资产。

## 2. 非目标

- 不恢复 Live2D SDK。
- 不在 MVP 阶段依赖本地 GPU 模型。
- 不把动画逻辑写进页面组件。
- 不追求实时面捕或复杂骨骼绑定。

## 3. 产品原则

AI Character Platform 的表现层应服务于角色本身，而不是变成单一数字人技术 Demo。

角色动起来的优先级：

1. 低成本可用：静态图也能产生状态表现。
2. 可配置：不同角色可覆盖默认状态动作。
3. 可降级：没有高级资产时仍可聊天、发声、展示状态。
4. 可扩展：后续可接入 ComfyUI 动图、视频、局部表情帧或远端 Avatar Provider。

## 4. 运行时分层

```text
Conversation / Studio
-> Runtime Event
-> Character Motion Runtime
-> Static Avatar View
-> Future Video / Sprite / Remote Avatar Adapter
```

### 4.1 Runtime Event

来源包括：

- LLM 流式文本事件。
- TTS started / chunk / done。
- ASR listening / transcribing。
- Workflow running / waiting_confirmation。
- error / interrupted。

### 4.2 Motion Runtime

Motion Runtime 只负责把状态和音频信号转换为视觉参数：

| 输入 | 说明 |
| --- | --- |
| `state` | `idle`、`listening`、`thinking`、`speaking` 等 |
| `mouthOpen` | 口型开合，来自 TTS/音量/viseme |
| `volume` | 当前播放音量 |
| `motionMap` | 角色级动作覆盖配置 |

输出：

| 输出 | 说明 |
| --- | --- |
| `expression` | 当前表情语义 |
| `motion` | 当前动作语义 |
| `intensity` | 动作强度 |
| `mouthOpen` | 归一化后的口型开合 |
| `cssVars` | 静态视图可直接消费的 CSS 变量 |

### 4.3 Static Avatar View

第一阶段使用 CSS transform 和 opacity 实现：

- idle：低强度呼吸。
- listening：轻微前倾和聚焦光圈。
- thinking / streaming / synthesizing：慢速浮动。
- speaking：口型、缩放、亮度与音量联动。
- interrupted：短促位移。
- error：低饱和、红色提示。

## 5. 角色配置

角色后续可在 `runtimeConfig` 或 Avatar Profile `config.runtime` 中覆盖：

```json
{
  "runtime": {
    "motionMap": {
      "speaking": {
        "expression": ["happy", "speaking"],
        "motion": ["talk-soft"]
      },
      "thinking": {
        "expression": "thinking",
        "motion": "float"
      }
    }
  }
}
```

v0.6 先使用语义动作名，不要求真实文件资产。后续如果接入视频或表情帧，可以把同一套语义映射到不同 adapter。
