# v0.7 Expression Asset Runtime

## 1. 目标

v0.7 目标是让角色表现层从“CSS 动作”升级到“状态驱动的表现资产”：

- 角色可为不同状态配置表情图、短视频或后续 ComfyUI 输出资产。
- Motion Runtime 根据 `state`、`expression`、`motion` 自动选择资产。
- 没有资产时继续使用 v0.6 的静态图片 + CSS 动作降级。
- 为后续 ComfyUI 表情帧、说话短视频、工作流动作触发打基础。

## 2. 非目标

- 不做实时面捕。
- 不要求本地 GPU。
- 不在前端直接调用 ComfyUI。
- 不把 ComfyUI workflow 绑死到单一页面。

## 3. 配置模型

角色配置继续写入 Avatar Profile `config.runtime`：

```json
{
  "runtime": {
    "motionMap": {
      "speaking": {
        "expression": ["happy", "speaking"],
        "motion": "talk-soft"
      }
    },
    "motionAssets": {
      "idle": {
        "kind": "image",
        "url": "/marketing/beautiful_girl.png"
      },
      "speaking": {
        "kind": "video",
        "url": "/avatars/emily/speaking.mp4",
        "posterUrl": "/avatars/emily/speaking.png",
        "loop": true
      }
    }
  }
}
```

## 4. 资产类型

| 类型 | 用途 | MVP 行为 |
| --- | --- | --- |
| `image` | 表情图、状态图、ComfyUI 输出图 | 作为当前头像图渲染 |
| `video` | 说话短视频、待机循环视频 | 使用 muted loop video 渲染 |
| `sprite` | 后续表情帧序列 | v0.7 只保留协议，不实现 |

## 5. Runtime 选择规则

1. 优先按当前 `state` 查找资产。
2. 如果同一状态有多个资产，优先匹配 `expressionCandidates`。
3. 其次匹配 `motionCandidates`。
4. 都没有匹配时取当前状态第一个资产。
5. 当前状态没有资产时，回退到头像主图。

## 6. 开发拆分

| 编号 | 任务 | 验收 |
| --- | --- | --- |
| M33 | Motion Asset Core | 有资产类型、normalize、resolve 和单测 |
| M34 | Runtime API 输出资产 | `/runtime` 和 `/preview` 返回 `motionAssets` / `motionAsset` |
| M35 | Static View 渲染资产 | 支持 image / video 资产覆盖主头像 |
| M36 | Avatar 配置入口 | 可在 Avatar 配置页编辑 `motionAssets` |
| M37 | ComfyUI 资产绑定 | 生成结果可作为角色表现资产绑定 |

## 7. 当前实现记录

- 已新增 `src/core/avatar-runtime/motion-assets.ts`，支持 `motionAssets` normalize 与 resolve。
- Motion Runtime 已按当前 `state`、`expressionCandidates`、`motionCandidates` 选择资产。
- `/runtime` 与 `/preview` 已透传 `motionAssets` 和当前 `motionAsset`。
- 静态头像视图已支持 image / video 资产覆盖主头像。
- 修正了 runtime motion 只在 state 匹配时复用，避免对话中使用陈旧 idle motion。

## 8. 后续方向

- ComfyUI workflow 输出多状态资产包。
- 说话短视频按 TTS 音频长度裁剪或循环。
- 工作流 step 触发指定 expression / motion。
- 资产预加载和缓存，减少对话切换时闪烁。
