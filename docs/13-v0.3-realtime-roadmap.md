# v0.3 Realtime Digital Human Roadmap

## 1. Current Assessment

v0.2 已经完成数字人核心闭环：

- LLM 对话流式返回。
- 百炼 CosyVoice TTS Provider 与缓存。
- AvatarProfile / AvatarAsset / AvatarGenerationJob 数据模型。
- Avatar 上传、生成、绑定、失败重试。
- Conversation 页面加载绑定 Avatar。
- 音频播放驱动 speaking 状态和 mouth openness。

当前主要短板：

- 语音输入仍是基础 ASR 占位能力，未形成实时语音对话。
- Chat、TTS、Avatar 状态还在单请求链路内串行执行，缺少 realtime session。
- 中断、barge-in、任务取消没有统一协议。
- 图片生成是本地静态占位 Provider，未接真实图片生成服务。
- WebRTC 只有接口边界，没有 session/token/signaling 实现。
- 缺少 Redis/queue 支撑短期状态、限流、任务恢复和多客户端同步。

## 2. v0.3 Goal

v0.3 目标是把 v0.2 的“文本驱动数字人”升级为“实时语音数字人”：

```text
Mic input
-> ASR partial/final transcript
-> LLM streaming
-> TTS streaming or low-latency chunks
-> Avatar realtime state
-> interrupt / barge-in
```

v0.3 不追求完整视频流数字人，不做复杂 Live2D/VRM 编辑器，不做多租户商业化后台。

## 3. Architecture Direction

### Runtime Session

新增 RealtimeSession 作为短生命周期状态容器：

- sessionId
- conversationId
- avatarProfileId
- status: idle | listening | transcribing | thinking | speaking | interrupted | closed
- activeMessageId
- activeAudioId
- createdAt / expiresAt

状态建议放 Redis，数据库只持久化会话、消息、音频结果和最终状态。

当前实现进度：

- 已新增 `REDIS_URL` 与 `REALTIME_SESSION_TTL_SECONDS`。
- 已新增 Redis-backed realtime session store。
- 已新增 session create / get / interrupt / close API。
- WebSocket events 尚未实现。

### Transport

第一版优先：

- Browser microphone: MediaRecorder / Web Audio。
- Client -> Server: WebSocket。
- Server -> Client: SSE 或 WebSocket events。

WebRTC 放在 v0.3 后半段：

- 先实现 token/session/signaling 边界。
- 不直接上完整音视频 SFU。
- 后续可接 LiveKit / Daily / Agora / 自建 WebRTC gateway。

### Redis Usage

本地开发可用 Docker 启 Redis：

```bash
docker compose up -d redis
```

用途：

- realtime session state
- short-lived event buffer
- ASR/TTS chunk cache
- rate limit
- generation job lock
- interrupt signal

## 4. API Draft

```text
POST /api/realtime/sessions
GET  /api/realtime/sessions/:sessionId
POST /api/realtime/sessions/:sessionId/interrupt
POST /api/realtime/sessions/:sessionId/close
WS   /api/realtime/sessions/:sessionId/events
```

Request:

```json
{
  "conversationId": "optional",
  "avatarProfileId": "optional",
  "knowledgeBaseId": "optional",
  "modelProviderId": "optional",
  "voiceProviderId": "optional"
}
```

Event examples:

```json
{ "type": "asr.partial", "text": "你好" }
{ "type": "asr.final", "text": "你好，介绍一下项目" }
{ "type": "llm.delta", "text": "这是" }
{ "type": "tts.chunk", "audioUrl": "/api/audio/..." }
{ "type": "avatar.state", "state": "speaking" }
{ "type": "session.interrupted", "reason": "user_barge_in" }
```

```text
rt:session:{sessionId}

```

## 6. Provider Work

### ASR Provider

新增统一接口：

```ts
interface StreamingASRProvider {
  transcribe(input: AsyncIterable<AudioChunk>): AsyncIterable<ASRChunk>;
}
```

第一版可支持：

- Browser Web Speech API fallback
- 阿里云 / OpenAI / 自定义 HTTP ASR
- mock ASR for tests

### TTS Streaming

当前 TTS 返回完整 audioUrl。v0.3 需要扩展：

- chunk events
- speech marks / viseme marks
- cancel signal
- cache by text + voice + format

### Avatar Runtime

v0.3 先做 runtime adapter：

- Static adapter: 当前 mouth openness。
- Live2D placeholder adapter。
- VRM placeholder adapter。

不在 v0.3 强制完成模型资产编辑器。

- latency metrics

## 8. Acceptance Definition

v0.3 完成时必须满足：

- 用户可以开启 realtime session。
- 麦克风输入可以产生 ASR final transcript。
- transcript 可以触发 LLM 回复。
- TTS 音频可以播放，Avatar 同步 speaking。
- 用户在 speaking 中再次说话可以 interrupt。
- interrupted message 有持久化状态。
- session 状态可查询，可关闭。
- Redis 不可用时能明确降级或返回可理解错误。
