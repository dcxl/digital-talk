# v0.5 开发落地拆分

## 1. 执行原则

v0.5 只做 AI Character Platform 核心闭环：

```text
Character
-> Appearance Asset
-> Scene
-> Memory
-> Workflow
-> Conversation Runtime
```

不继续扩展 Live2D、VRM、动作系统和复杂 WebRTC。

工程约束：

- 页面、业务逻辑、API 请求、hooks、types 分文件。
- 不继续扩大历史大文件。
- 先做服务端数据闭环，再做页面。
- 每个阶段必须能 build/test。

## 2. 阶段顺序

| 阶段 | 目标 | 依赖 |
| --- | --- | --- |
| M36 | Character 数据模型与 migration | 现有 Prisma |
| M37 | Character API 与 service | M36 |
| M38 | Character Library / Studio UI | M37 |
| M39 | Scene Binding | M36 / M37 |
| M40 | Character Memory MVP | M36 / M37 |
| M41 | Workflow MVP | M36 / M37 |
| M41.5 | ComfyUI Provider 边界 | M36 / M37 |
| M42 | 验收、清理、兼容检查 | 全部 |

## 3. M36 Character 数据模型

### 任务

- 更新 `prisma/schema.prisma`。
- 新增 Character 相关枚举。
- 新增表：
  - `CharacterProfile`
  - `CharacterScene`
  - `CharacterSceneBinding`
  - `CharacterMemory`
  - `CharacterWorkflow`
  - `CharacterWorkflowExecution`
- `Conversation` 增加：
  - `characterId`
  - `sceneId`
- 生成 Prisma migration。

### 文件

```text
prisma/schema.prisma
prisma/migrations/*
```

### 验收

- `npx prisma generate` 通过。
- `npx prisma migrate dev` 通过。
- 旧 `AvatarProfile`、`Conversation` 查询不报错。

## 4. M37 Character API 与 Service

### 任务

- 新增 Character service。
- 新增 Character API。
- API 返回统一 envelope。
- 旧 Avatar 外观可被 Character 引用。

### 文件

```text
src/services/characters/
  repository.ts
  presenter.ts
  schema.ts

src/app/api/characters/
  route.ts

src/app/api/characters/[characterId]/
  route.ts
```

### API

- `GET /api/characters`
- `POST /api/characters`
- `GET /api/characters/:characterId`
- `PATCH /api/characters/:characterId`
- `DELETE /api/characters/:characterId`

### 验收

- 可创建 draft Character。
- 可绑定 `appearanceProfileId`。
- 可软删除 Character。
- 返回数据不暴露敏感 provider 信息。

## 5. M38 Character Library / Studio UI

### 任务

- `/characters` 独立使用 Character 数据，不再直接复用 Avatar 管理 hook。
- 保留 `/avatar` 兼容入口。
- 新增 Character List、Profile、Appearance、Voice、Scene、Memory、Workflow 面板。

### 文件

```text
src/features/workspace/pages/characters-page.tsx
src/features/workspace/characters/
  character-list.tsx
  character-profile-form.tsx
  character-appearance-panel.tsx
  character-voice-panel.tsx
  character-scene-panel.tsx
  character-memory-panel.tsx
  character-workflow-panel.tsx
  use-character-management.ts
  types.ts
  constants.ts
```

### 验收

- 角色库可显示、搜索、选择角色。
- 可新建和编辑基础信息。
- 页面不直接调用 Prisma。
- 页面文案全部中文。

## 6. M39 Scene Binding

### 任务

- 新增 scene service。
- 新增 scene API。
- 角色可绑定场景，且可设置默认场景。
- 创建会话时写入 `sceneId`。

### API

- `GET /api/scenes`
- `POST /api/scenes`
- `PATCH /api/scenes/:sceneId`
- `POST /api/characters/:characterId/scenes/:sceneId/bind`
- `DELETE /api/characters/:characterId/scenes/:sceneId/bind`

### 验收

- 同一角色最多一个默认场景。
- 未绑定场景时仍可普通对话。

## 7. M40 Character Memory MVP

### 任务

- 新增 memory service。
- 新增 memory API。
- Chat runtime 注入 active memory。
- 暂不做自动写入，只做手动管理。

### API

- `GET /api/characters/:characterId/memories`
- `POST /api/characters/:characterId/memories`
- `PATCH /api/characters/:characterId/memories/:memoryId`
- `DELETE /api/characters/:characterId/memories/:memoryId`

### 验收

- 可新增、禁用、删除记忆。
- deleted/disabled/expired 记忆不会注入 prompt。

## 8. M41 Workflow MVP

### 任务

- 新增 workflow service。
- 新增 workflow API。
- 支持手动运行和执行日志。
- 需要确认的 workflow 不自动执行。

### API

- `GET /api/characters/:characterId/workflows`
- `POST /api/characters/:characterId/workflows`
- `PATCH /api/characters/:characterId/workflows/:workflowId`
- `POST /api/characters/:characterId/workflows/:workflowId/run`

### 验收

- 可创建 workflow。
- 可手动执行。
- 有 execution 记录。
- failed 状态有错误信息。

## 9. M41.5 ComfyUI Provider Boundary

### 任务

- 新增 ComfyUI 配置读取。
- 新增 provider interface。
- 新增 appearance generation API。
- 先支持 mock / disabled 状态。
- 真实 ComfyUI 调用后续接入。

### 文件

```text
src/services/comfyui/
  provider.ts
  config.ts
  workflow.ts

src/app/api/characters/[characterId]/appearance-generations/
  route.ts
```

### 验收

- 未配置 ComfyUI 时返回 `provider_not_configured`。
- mock provider 可生成 job 记录。
- 不允许前端提交任意 workflow JSON。

## 10. M42 清理与验收

### 任务

- 移除无用 Live2D 前端 runtime 和 API。
- 移除 `pixi-live2d-display` / `pixi.js` 依赖。
- 保留历史文档，但标记为 historical。
- 校验 `/avatar` 静态外观兼容。
- 运行测试与 build。

### 验收命令

```bash
npm run test
npm run lint
npm run build
```
