# v0.4 Avatar Runtime Roadmap

## 1. Current Assessment

v0.3 已经完成“实时语音数字人”的基础闭环：

- Realtime session：Redis session store、event buffer、create/get/interrupt/close API。
- ASR：Streaming ASR provider、百炼实时 ASR adapter、OpenAI-compatible ASR、mock ASR。
- Conversation：麦克风 -> realtime ASR -> conversation user message -> chat 复用 `userMessageId`。
- TTS：百炼 CosyVoice WebSocket chunk、`tts.chunk` / `tts.done` 事件、前端渐进播放。
- Interrupt：前端 barge-in、服务端 abort 状态修正、interrupted message 持久化。
- Avatar：Static runtime adapter，Live2D / VRM profile 可保存并通过 placeholder adapter 降级。
- 验收：`npm run smoke:v0.3`、desktop/mobile QA、失败恢复测试。

当前短板已经从“实时语音链路”转移到“真实数字人表现层”：

- Live2D / VRM 仍是占位 runtime，没有真实模型加载、渲染、动作、表情。
- AvatarAsset 具备 `image/live2d/vrm` 类型，但缺少模型包 manifest、文件校验、入口文件解析。
- mouth openness 仍以音量驱动为主，没有 speech marks / viseme mapping。
- TTS chunk 已预留 `marks` 字段，但 provider 到 Avatar runtime 的口型数据链路未闭合。
- 图片生成 provider 仍是静态占位，不能稳定生成生产可用的角色资产。
- WebSocket events 和 WebRTC 未完成，但此时直接做 WebRTC 会放大复杂度，无法解决 Avatar 仍是占位的问题。

结论：v0.4 应优先把数字人从“静态图片 + 状态动画”升级为“可加载、可驱动、可降级的 Avatar Runtime”。WebRTC 放到 v0.5 或 v0.4 后置探索。

## 2. v0.4 Goal

v0.4 目标：

```text
Avatar asset pack
-> manifest validation
-> runtime adapter
-> canvas/avatar renderer
-> TTS marks or audio-level mouth sync
-> conversation state mapping
-> runtime smoke + visual QA
```

必须交付：

- 支持上传并绑定 Live2D 模型包。
- Conversation 页面可以真实渲染 Live2D Avatar，不再只是 placeholder icon。
- Avatar state 可驱动待机、聆听、思考、说话、中断、错误等表现。
- 口型同步先支持 volume fallback，再接入 speech marks / viseme 数据。
- Static / Live2D / VRM runtime 都有明确降级路径。
- 新增 v0.4 smoke，能验证 asset manifest、runtime snapshot、页面渲染、fallback。

不作为 v0.4 强制目标：

- 完整 Live2D 编辑器。
- 完整 VRM 动作系统。
- WebRTC SFU / 低延迟音视频网关。
- AI 视频级真人数字人生成。
- 多租户商业化资产市场。

## 3. Architecture Direction

### Avatar Runtime Layer

现有 `AvatarProvider.getRuntime()` 已返回 runtime snapshot。v0.4 需要把它扩展成可驱动前端 renderer 的 runtime contract。

建议 runtime snapshot 包含：

```ts
interface AvatarRuntimeSnapshot {
  driver: "static" | "live2d" | "vrm";
  status: "ready" | "placeholder" | "degraded" | "error";
  adapterId: string;
  adapterName: string;
  asset?: {
    id: string;
    type: "image" | "live2d" | "vrm";
    entrypoint: string;
    manifestUrl?: string;
    files?: Array<{
      path: string;
      mimeType: string;
      url: string;
    }>;
  };
  capabilities: {
    image: boolean;
    live2d: boolean;
    vrm: boolean;
    viseme: boolean;
    motions: boolean;
    expressions: boolean;
  };
  mouth: {
    openness: number;
    source: "audio-volume" | "viseme" | "speech-mark" | "none";
  };
  motion?: {
    name: string;
    loop?: boolean;
  };
  expression?: {
    name: string;
    intensity?: number;
  };
}
```

### Frontend Renderer Split

保持工程化拆分：

```text
src/features/avatar-runtime/
  components/
    avatar-runtime-stage.tsx
    live2d-canvas.tsx
    static-avatar-view.tsx
    vrm-canvas-placeholder.tsx
  hooks/
    use-avatar-runtime.ts
    use-mouth-sync.ts
    use-runtime-motion.ts
  lib/
    manifest.ts
    mouth.ts
    renderer-state.ts
  types.ts
```

Conversation 页面只接入 `AvatarRuntimeStage`，不直接处理 Live2D SDK 细节。

### Asset Manifest

Live2D 上传建议先支持 zip 包或目录式上传，入库时生成 manifest：

```json
{
  "driver": "live2d",
  "entrypoint": "model.model3.json",
  "textures": ["textures/texture_00.png"],
  "motions": {
    "idle": ["motions/idle.motion3.json"],
    "speaking": ["motions/speaking.motion3.json"]
  },
  "expressions": {
    "neutral": "expressions/neutral.exp3.json",
    "happy": "expressions/happy.exp3.json"
  },
  "validatedAt": "2026-07-02T00:00:00.000Z"
}
```

可先把 manifest 写入 `AvatarAsset.metadata.runtime`，不急于新增表。只有当需要逐文件权限、版本和差异更新时，再新增 `AvatarAssetFile`。

### Mouth Sync

v0.4 分两级：

1. Volume fallback：沿用当前 audio analyser，驱动 `mouth.openness`。
2. Speech marks / viseme：当 TTS provider 返回 marks 时，按时间轴驱动口型。

建议映射：

```text
tts.chunk.marks
-> normalizeSpeechMarks()
-> mouth timeline
-> useMouthSync()
-> AvatarRuntimeStage
-> Live2D parameter ParamMouthOpenY
```

第一版只需要口型开合，不做完整音素表情。

## 4. API Draft

### Avatar Asset Upload

```text
POST /api/avatar-assets/upload
```

扩展字段：

```json
{
  "type": "live2d",
  "profileId": "optional",
  "entrypoint": "model.model3.json"
}
```

返回：

```json
{
  "asset": {
    "id": "asset_1",
    "type": "live2d",
    "status": "active",
    "metadata": {
      "runtime": {
        "driver": "live2d",
        "entrypoint": "model.model3.json",
        "validated": true
      }
    }
  }
}
```

### Runtime Snapshot

```text
GET /api/avatar-profiles/:avatarId/runtime
POST /api/avatar-profiles/:avatarId/preview
```

`preview` 保留现有接口，新增 `runtime` 字段继续作为前端预览来源。`runtime` GET 用于 Conversation 页面加载当前 profile runtime。

### Runtime Events

现有 chat stream 已有：

```text
avatar.state
tts.chunk
tts.done
```

v0.4 可新增：

```text
avatar.runtime.ready
avatar.runtime.degraded
avatar.mouth
avatar.motion
avatar.expression
```

不强制在第一步全部落地。优先让 renderer 能根据现有 `avatar.state` 和音频状态工作。

## 5. Data Model Direction

优先复用现有模型：

- `AvatarProfile.driver`
- `AvatarProfile.config`
- `AvatarAsset.type`
- `AvatarAsset.storageKey`
- `AvatarAsset.publicUrl`
- `AvatarAsset.metadata`

建议 `AvatarAsset.metadata.runtime`：

```json
{
  "runtime": {
    "driver": "live2d",
    "entrypoint": "model.model3.json",
    "manifestVersion": 1,
    "files": [
      {
        "path": "model.model3.json",
        "mimeType": "application/json",
        "size": 1234
      }
    ],
    "validation": {
      "ok": true,
      "errors": []
    }
  }
}
```

需要新增表的条件：

- 同一模型包支持多版本。
- 单文件级权限或签名 URL。
- 模型包增量更新。
- 资产市场或多人协作。

在这些需求出现前，不新增 `AvatarAssetFile`，避免 v0.4 过度建模。

## 6. Provider Work

### Avatar Provider

扩展 `AvatarProvider`：

```ts
interface AvatarProvider {
  getRuntime(input: AvatarRuntimeInput): Promise<AvatarRuntimeResult>;
  validateAsset?(input: AvatarAssetValidationInput): Promise<AvatarAssetValidationResult>;
}
```

Provider 责任：

- 根据 driver 和 asset metadata 返回 runtime snapshot。
- Live2D asset 缺失时降级 static。
- manifest 不合法时返回 `status=degraded/error`。
- 不负责前端 canvas 渲染细节。

### Image Generation Provider

v0.4 可以补真实图片生成 provider，但它是 P1，不应阻塞 Live2D runtime。

最低要求：

- 保留 static provider，避免成本。
- 新增真实 provider adapter 时必须走 `AvatarGenerationJob`。
- 生成结果仍进入 `AvatarAsset`。
- 失败可重试，不能阻塞已有 profile。

## 7. Milestones

### M21 Project Audit And Runtime Contract

- 对现有 AvatarProfile / AvatarAsset / runtime adapter 做差距审计。
- 定义 `AvatarRuntimeSnapshot` / asset manifest 类型。
- 新增 runtime endpoint 或扩展 preview runtime contract。
- 补 runtime contract 单元测试。

### M22 Live2D Asset Pack Upload

- 支持 Live2D zip 上传或 manifest 上传。
- 校验 `model3.json` 入口、texture、motion、expression 文件。
- 将 manifest 写入 `AvatarAsset.metadata.runtime`。
- Avatar 配置页可选择并绑定 Live2D asset。
- 错误提示必须中文可理解。

### M23 Live2D Renderer MVP

- 新增独立 `features/avatar-runtime`。
- Conversation / Avatar preview 引入 `AvatarRuntimeStage`。
- Live2D asset ready 时渲染 canvas。
- 无 asset 或加载失败时自动降级 static placeholder。
- 桌面/移动无横向溢出，canvas 非空。

### M24 Mouth Sync Timeline

- 抽象 `MouthSyncSource`。
- 支持 audio volume fallback。
- 支持 `tts.chunk.marks` / `tts.done.marks` 转 timeline。
- Live2D `ParamMouthOpenY` 由统一 hook 驱动。
- 没有 marks 时不报错，继续使用音量。

### M25 Motion And Expression Mapping

- 定义 state -> motion/expression 映射：
  - idle -> idle motion
  - listening -> attentive / neutral
  - thinking -> blink / thinking
  - speaking -> speaking motion + mouth
  - interrupted -> surprised / reset
  - error -> sad / neutral fallback
- 支持 profile.config 覆盖映射。
- Avatar 配置页预览待机、思考、说话、中断、错误。

### M26 Runtime Observability And Recovery

- runtime load error 进入 UI 状态和 API response。
- 记录 runtime driver、assetId、fallbackDriver、loadLatencyMs。
- Playground 或 Avatar 页面展示 runtime snapshot。
- 增加 asset 缺失、manifest 错误、renderer fallback 测试。

### M27 v0.4 Acceptance

- 新增 `npm run smoke:v0.4`。
- 覆盖 static / live2d runtime snapshot。
- 覆盖 Live2D asset manifest validation。
- 覆盖 Conversation 页面 Live2D 渲染基本检查。
- 覆盖 fallback：asset 缺失、manifest 无效、driver 不支持。
- 保留 `npm run smoke:v0.3` 全链路通过。

## 8. Acceptance Definition

v0.4 完成时必须满足：

- 用户可以上传或绑定 Live2D 模型资产。
- AvatarProfile 使用 `driver=live2d` 时，Conversation 页面渲染真实 canvas/runtime，而不是 placeholder icon。
- TTS 播放时 Avatar 有可见口型变化。
- 无 speech marks 时使用音量 fallback；有 speech marks 时优先使用 marks。
- Runtime 加载失败时自动降级并给出可理解错误。
- Static / Live2D 都能通过 runtime snapshot API 查询。
- Avatar 配置页可以预览 idle / thinking / speaking / interrupted / error。
- v0.4 smoke、lint、test、build 通过。

10. 新增 `smoke:v0.4` 与浏览器 canvas QA。

## 10. v0.5 Preview

v0.4 完成真实 Avatar Runtime 后，v0.5 再考虑：

- WebSocket events 替代或补充 SSE。
- WebRTC token/session/signaling。
- 服务端级 TTS chunk cancel。
- VRM renderer MVP。
- 真实图片生成 provider。
- Tool calling 与动作/表情联动。
