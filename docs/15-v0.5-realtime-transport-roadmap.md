# v0.5 Realtime Transport Roadmap

> 文档状态：历史运行时路线。本文已被 `18-v0.5-character-platform-core.md` 取代，不再作为当前 v0.5 开发主线；仅保留用于后续 Realtime Transport 设计追溯。

## 1. Current Baseline

v0.4 已完成 Avatar Runtime Core：

- Live2D package manifest 校验。
- `/api/avatar-profiles/:avatarId/runtime` runtime snapshot。
- Conversation / Avatar 页面真实渲染 Live2D。
- mouth-sync timeline，支持 `tts.chunk.marks` / `tts.done.marks`。
- state -> motion/expression mapping。
- runtime diagnostics 与 fallback。
- `npm run smoke:v0.4`。

v0.5 的重点不再是继续堆页面，而是把实时链路收敛成稳定、可取消、可扩展的 runtime transport。

## 2. Goal

v0.5 目标：

```text
Realtime session
-> typed runtime event envelope
-> SSE / WebSocket transport abstraction
-> client transport switch
-> barge-in command
-> LLM / TTS / audio queue cancellation
-> smoke + recovery tests
```

必须交付：

- Runtime events 使用统一 envelope，支持 sessionId、sequence、timestamp、event type、payload。
- 前端不直接绑定 SSE，实现 transport abstraction。
- WebSocket 作为可选 transport，SSE 保留 fallback。
- Barge-in 时能停止前端音频、清理 TTS chunk 队列、标记 session interrupted。
- 服务端已有可取消点必须接入 `AbortSignal`。
- 新增 v0.5 smoke，覆盖 transport contract、interrupt recovery、fallback。

不作为 v0.5 强制目标：

- 完整 WebRTC SFU。
- 视频流数字人。
- VRM renderer。
- Tool Calling 动作系统。
- 本地大模型推理。

## 3. Architecture Direction

### Runtime Event Envelope

新增统一事件信封：

```ts
interface RealtimeEventEnvelope<TType extends string, TPayload> {
  id: string;
  sessionId: string;
  sequence: number;
  type: TType;
  payload: TPayload;
  timestamp: string;
}
```

现有 `RuntimeEvent` 继续服务 `/api/chat` SSE。v0.5 的 `RealtimeEventEnvelope` 用于 session transport，不直接替换 chat stream。

### Transport Abstraction

前端增加：

```text
src/features/digital-human/realtime-transport/
  event-envelope.ts
  realtime-transport.ts
  sse-transport.ts
  websocket-transport.ts
  use-realtime-transport.ts
```

要求：

- Conversation shell 只调用 transport hook。
- SSE / WebSocket 的解析逻辑独立。
- WebSocket 不可用时自动 fallback SSE。
- 所有 transport 都输出同一种 envelope。

### Command Events

v0.5 不只下发事件，也需要上行命令：

```text
client.command.interrupt
client.command.audio_chunk
client.command.close
```

第一版可以先实现 interrupt command，audio chunk 后置。

### Cancellation Chain

barge-in 必须按顺序执行：

```text
user starts speaking
-> stop audio element
-> clear tts chunk queue
-> abort active chat request
-> POST /api/chat/interrupt or realtime command
-> mark assistant message interrupted
-> avatar state interrupted/listening
```

服务端继续生成但前端已停止播放不算完整完成。v0.5 需要逐步接入 provider cancel。

## 4. API Draft

### Existing SSE Replay

继续保留：

```text
GET /api/realtime/sessions/:sessionId/events
```

### WebSocket Transport

建议目标：

```text
WS /api/realtime/sessions/:sessionId/ws
```

注意：Next.js App Route 对 WebSocket upgrade 支持有限。实现可以选择：

1. 独立 Node WebSocket server。
2. Next custom server。
3. 开发期先用 SSE transport abstraction，WebSocket adapter 后接真实 server。

v0.5 第一阶段先完成 contract 与前端抽象，避免卡在部署形态。

### Interrupt Command

继续复用：

```text
POST /api/realtime/sessions/:sessionId/interrupt
POST /api/chat/interrupt
```

后续 WebSocket 可发送：

```json
{
  "type": "client.command.interrupt",
  "payload": {
    "reason": "barge_in"
  }
}
```

## 5. Milestones

### M28 Transport Contract

- 新增 `RealtimeEventEnvelope` 类型与 builder。
- 统一 sequence/id/timestamp 生成。
- 为现有 realtime event buffer 增加 envelope 转换工具。
- 单元测试覆盖 sequence、payload、类型守卫。

当前实现进度：

- 已新增 `src/core/realtime/event-envelope.ts`。
- 已支持 envelope builder、stored event -> envelope 转换、类型守卫。
- 已新增 envelope 单元测试。

### M29 Frontend Transport Abstraction

- 新增 `realtime-transport` feature 模块。
- SSE transport 读取现有 `/events` endpoint。
- WebSocket transport 先实现接口和不可用 fallback。
- `useVoiceInput` / shell 不直接依赖具体 transport。

### M30 Interrupt Cancellation Chain

- 前端 barge-in 清理 audio queue、mouth timeline、active chat abort。
- 后端 chat interrupt 返回可追踪状态。
- interrupted assistant message 持久化保持一致。

### M31 Streaming TTS Cancel

- TTS provider stream 接入 `AbortSignal`。
- 前端停止播放时清理 queued chunks。
- 服务端停止继续 enqueue `tts.chunk`。
- smoke 覆盖 TTS cancel 后不继续 speaking。

### M32 WebSocket Server MVP

- 选择部署形态：独立 Node server 或 Next custom server。
- 实现 session event subscribe。
- 支持 `client.command.interrupt`。
- SSE fallback 仍可用。

### M33 Realtime ASR Chunk Transport

- WebSocket audio chunk 协议。
- ASR partial/final envelope。
- 失败恢复和 session close 清理。

### M34 v0.5 Acceptance

- 新增 `npm run smoke:v0.5`。
- 覆盖 envelope contract。
- 覆盖 SSE fallback。
- 覆盖 interrupt cancellation chain。
- 覆盖 WebSocket command，如果当前环境启用 WebSocket。
- 保留 `npm run smoke:v0.4` 通过。

## 6. Acceptance Definition

v0.5 完成时必须满足：

- 前端具备 transport abstraction，不再把 realtime 能力硬编码到页面组件。
- SSE fallback 可用。
- WebSocket transport 有明确实现或降级路径。
- barge-in 后音频、TTS queue、chat request、avatar state 一致清理。
- 服务端可取消点接入 `AbortSignal`。
- `lint`、`test`、`build`、`smoke:v0.4`、`smoke:v0.5` 通过。

## 7. Recommended First Tasks

3. 新增 `features/digital-human/realtime-transport` 类型层。

5. 再进入 interrupt cancellation chain。
