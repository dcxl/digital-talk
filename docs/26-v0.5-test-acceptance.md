# v0.5 测试与验收

## 1. 总体验收

v0.5 完成标准：

- Character 是一等对象。
- `/characters` 是主入口。
- `/avatar` 仍可兼容静态外观配置。
- Conversation 可持久化 `characterId` / `sceneId`。
- Character 可管理 Scene、Memory、Workflow。
- ComfyUI Provider 边界存在，但真实调用可后置。
- Live2D / VRM 不再进入新流程。

## 2. 必跑命令

```bash
npm run test
npm run lint
npm run build
```

涉及 schema 时追加：

```bash
npx prisma generate
npx prisma migrate dev
```

## 3. 数据模型验收

覆盖：

- CharacterProfile CRUD。
- CharacterScene CRUD。
- CharacterSceneBinding unique 约束。
- CharacterMemory 软删除。
- CharacterWorkflowExecution 状态流转。
- Conversation 可保存 `characterId` / `sceneId`。

建议测试文件：

```text
src/services/characters/repository.test.ts
src/services/character-scenes/repository.test.ts
src/services/character-memories/repository.test.ts
src/services/character-workflows/repository.test.ts
```

## 4. API 验收

必须验证：

| API | 验收 |
| --- | --- |
| `GET /api/characters` | 返回分页列表 |
| `POST /api/characters` | 可创建 draft |
| `PATCH /api/characters/:id` | 可更新基础字段 |
| `DELETE /api/characters/:id` | 软删除 |
| `POST /api/characters/:id/scenes/:sceneId/bind` | 可绑定默认场景 |
| `POST /api/characters/:id/memories` | 可新增手动记忆 |
| `POST /api/characters/:id/workflows` | 可新增 workflow |
| `POST /api/characters/:id/appearance-generations` | 未配置 ComfyUI 时返回明确错误 |

## 5. 页面验收

`/characters`：

- 空状态可新建角色。
- 列表可选择角色。
- 表单可保存基础信息。
- Appearance 面板可上传或绑定静态图片。
- Scene 面板可绑定默认场景。
- Memory 面板可新增、禁用、删除。
- Workflow 面板可创建、手动执行。

`/avatar`：

- 仍可访问。
- 静态 avatar 预览不报错。
- 不展示 Live2D / VRM 作为新建推荐。

## 6. Runtime 验收

Chat：

- 新会话可带 `characterId`。
- 已绑定 Character 的会话继续聊天不丢失绑定。
- Memory 只注入 active 且未过期的记录。
- Scene prompt 能进入 runtime 上下文。
- TTS 失败不影响文本回复。
- ASR 失败不影响文本输入。

## 7. ComfyUI 验收

未启用：

- API 返回 `provider_not_configured`。
- UI 不允许点击真实生成。

Mock：

- 可以创建 generation job。
- job 可以完成并返回静态 asset。

真实 ComfyUI：

- 可以提交 workflow。
- 可以轮询结果。
- 输出图片落库为资产。
- 资产可绑定到 Character。

## 8. 清理验收

Live2D / VRM 退场后：

- `npm run build` 不再出现 Live2D 相关 trace warning。
- `package.json` 不再包含 `pixi-live2d-display` / `pixi.js`。
- 没有前端动态 import Live2D SDK。
- 没有 `/api/avatar-runtime/live2d-packages` 新业务依赖。

## 9. 手动 Smoke

建议顺序：

1. 启动 PostgreSQL。
2. 运行 migration。
3. 启动 dev server。
4. 打开 `/characters`。
5. 创建角色。
6. 上传或绑定一张图片。
7. 绑定场景。
8. 新增一条记忆。
9. 创建一个手动 workflow。
10. 从角色进入对话。
11. 确认回答包含角色设定。
